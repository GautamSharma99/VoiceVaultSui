# Pay During HTTP Request

# High-Level Architecture

```text
Frontend / SDK Client
↓
x402 Client Middleware
↓
VoiceVault API Gateway
↓
402 Payment Required
↓
x402 Facilitator (Sui)
↓
Sui Settlement
↓
Voice Access Granted
↓
LiveKit / TTS Runtime
↓
Revenue Split + Usage Tracking
```

x402 servers return `402 Payment Required`, clients resend with payment proof, and a facilitator verifies + settles the payment without holding funds. Facilitators expose verify/settle flows and Sui is listed as a supported exact-payment network. ([Cloudflare Docs][2])

# 2. HTTP 402 Response System

If unpaid:

server returns:

```http
402 Payment Required
```

with payment metadata:

```json
{
  "price": "0.20 USDC",
  "network": "sui",
  "merchant_address": "0xplatform",
  "resource": "/api/tts/generate",
  "voice_id": "voice_123"
}
```

This matches x402 flow where the server provides price, token, network, and merchant destination. ([Cloudflare Docs][2])

---

# 3. Client Retry with Payment Proof

Client:

* signs payment
* attaches proof header

example:

```http
PAYMENT-SIGNATURE
```

and retries request.

Server verifies and serves resource. ([Cloudflare Docs][2])

---

# 4. Sui-Native Facilitator

Need:

# VoiceVault Facilitator

Responsibilities:

* verify payment payload
* settle transaction on Sui
* confirm payment legitimacy
* prevent replay attacks
* enforce usage limits

Can use:

## Sui x402 Facilitator Layer

instead of building settlement manually.

Facilitator verifies and broadcasts pre-signed transactions and should not custody funds. ([Cloudflare Docs][2])
